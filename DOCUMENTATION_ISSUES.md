### 1.1. Метод `htmlToPlainTextForJson`

**Место в README:** Строки 133-138, раздел "Метод `htmlToPlainTextForJson`"

**Описание:**
Пример показывает неверный JSON-вывод метода. В README показан результат Java-строки, а не фактический вывод метода.

**README (строки 131-138):**
```java
String html = "<p>Привет<br/>мир!</p>";
String json = HtmlToText.htmlToPlainTextForJson(html);
// Результат: "Привет\nмир!"  // ← НЕВЕРНО
```

**Фактический вывод (из E2E test `test-simple-paragraph.exp-output`):**
```
"Это простой абзац с текстом."  // ← Без \n в конце
```

**Фактический вывод для `<br>` (из E2E test `test-line-breaks.exp-output`):**
```
"Текст с\nпереносом строки внутри\nТекст в div с\n\nдвумя переносами"
```

**Проблема:**
1. Метод экранирует `\n` как `\\n` (двойное экранирование для JSON)
2. В README показан необработанный `\n` вместо `\\n`
3. Нет кавычек вокруг результата в примере (хотя метод возвращает строку в кавычках)

**Ожидаемое исправление:**
```java
String json = HtmlToText.htmlToPlainTextForJson(html);
// Результат: "Текст с\\nпереносом"  // ← Двойное экранирование
```

**Статус:** 🔴 Требует исправления

### 1.3. Сохранение `&nbsp;` (неразрывных пробелов)

**Место в README:** Раздел 4.4 "Нормализация текста", строки 588-608

**Описание:**
В README утверждается, что `&nbsp;` сохраняется, но на самом деле он удаляется regex-паттерном.

**README (строки 588-608):**
```
#### normalizeSpaces(String text)

```regex
[ \t\r\n]+ → " "
```

Удаляет только невидимые пробелы (пробелы, табуляция, переносы строк), заменяя их одним пробелом. Unicode-символы пробела сохраняются, включая:
- `&nbsp;` (`\u00A0`) — неразрывный пробел
- `&emsp;` (`\u2003`) — эм-пробел  
- `&ensp;` (`\u2002`) — эм-полупробел

Это позволяет сохранить видимую вёрстку при конвертации в текст.

**Примеры:**

```
"Текст&nbsp;с&nbsp;неразрывным" → "Текст\u00A0с\u00A0неразрывным"
"Текст\tс\tтабуляцией" → "Текст с табуляцией"
```
```

**Фактический код (строки 227-229 HtmlToText.java):**
```java
private static String normalizeSpaces(String text) {
    return text.replaceAll("[ \t\r\n]+", " ");
}
```

**Фактический вывод (из E2E test `test-nbsp-preservation.exp-output`):**
```
"Текст с неразрывными пробелами"
```

**Проверка байтов (od -c):**
```
0000000   " 320 242 320 265 320 272 321 201 321 202     321 201     320
0000020 275 320 265 321 200 320 260 320 267 321 200 321 213 320 262 320
...
```

**Проблема:**
1. Regex `[ \t\r\n]+` включает обычный пробел (ASCII 0x20 = 32 в octal)
2. `&nbsp;` (`\u00A0`, 0xC2 0xA0 в UTF-8) НЕ сохраняется, а удаляется
3. В README утверждается обратное — это критическая ошибка документации

**Фактическое поведение:**
- ASCII пробелы (0x20), табуляция (0x09), `\r` (0x0D), `\n` (0x0A) → удаляются
- `&nbsp;` (`\u00A0`) → удаляется как часть текста
- `&emsp;`, `&ensp;` → удаляются

**Ожидаемое исправление:**
Либо исправить код для сохранения `&nbsp;`, либо исправить README для отражения фактического поведения.

**Ожидаемый текст в README:**
```
#### normalizeSpaces(String text)

```regex
[ \t\r\n]+ → " "
```

Удаляет невидимые ASCII-пробелы (пробелы, табуляция, переносы строк), заменяя их одним пробелом.
Все Unicode-символы пробела, включая `&nbsp;` (`\u00A0`), **удаляются** вместе с остальным текстом.

**Примеры:**

```
"Текст&nbsp;с&nbsp;неразрывным" → "Текст с неразрывным"  // ← &nbsp; удален
"Текст\tс\tтабуляцией" → "Текст с табуляцией"
"Текст    с    пробелами" → "Текст с пробелами"
```
```

**Статус:** 🔴 Критическая ошибка — требует немедленного исправления
